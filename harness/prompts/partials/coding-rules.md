## Coding Rules
- 불변성을 우선한다. 객체/배열 직접 변경을 피한다.
- 가능한 한 순수 함수를 선호한다.
- 깊은 중첩 대신 early return을 사용한다.
- 매직 넘버/문자열은 상수로 분리한다.
- 이름은 역할이 드러나야 한다. 축약어와 모호한 이름을 피한다.
- 상속보다 조합을 우선한다.
- 미래를 과도하게 예측한 추상화는 만들지 않는다.
- 공개 경계의 입력/출력 타입은 명확히 정의한다.
- 외부 입력은 항상 검증한다.
- 에러를 삼키지 않는다.
- 사용자 입력 오류, 비즈니스 오류, 외부 시스템 오류를 구분한다.
- 사용자 메시지와 내부 로그를 분리한다.
