# /commit

작업이 완료된 변경사항을 git에 커밋하고 원격 저장소에 푸시합니다.

## 실행 절차

1. `git status`로 변경된 파일 목록을 확인한다.
2. 변경사항이 없으면 "커밋할 변경사항이 없습니다." 를 출력하고 종료한다.
3. `git diff`로 변경 내용을 파악한다.
4. `git log --oneline -5`로 최근 커밋 메시지 스타일을 확인한다.
5. 변경 내용을 바탕으로 커밋 메시지 초안을 작성해 사용자에게 보여준다.
6. 사용자 확인 후 다음을 실행한다:
   - `git add` — 관련 파일만 명시적으로 스테이징 (`.env`, 시크릿, 대용량 바이너리 제외)
   - `git commit -m "..."` — 확정된 메시지로 커밋
   - `git push` — 원격 저장소에 푸시
7. 완료 후 `git log --oneline -3`으로 결과를 확인한다.

## 커밋 메시지 규칙

- Conventional Commits 형식 사용: `type: 설명`
- type: `feat`, `fix`, `docs`, `refactor`, `chore` 중 선택
- 설명은 한국어로 작성 (이 프로젝트 관행)
- 변경의 "무엇"보다 "왜"를 중심으로 작성

## 주의사항

- `.env`, API 키, 비밀번호 등 민감 정보가 포함된 파일은 절대 스테이징하지 않는다.
- `git add -A` 또는 `git add .` 대신 파일을 명시해 스테이징한다.
- 사용자 확인 없이 force push하지 않는다.
- main/master 브랜치에 force push 요청이 오면 경고 후 거부한다.
